# TG 批量私信系统 v1.3

简单实用的 Telegram 批量私信工具，支持账号管理、私信广告、用户采集三大功能。

## ✨ 功能特点

### 📂 账号管理
- ✅ 批量导入 Session 文件
- ✅ 检测账号状态
- ✅ 显示详细信息（用户名/手机/状态/最后登录）
- ✅ 全选/清空/删除失效账号
- ✅ 实时统计

### ✉️ 私信广告
- ✅ 批量发送文本消息
- ✅ 支持变量替换（{username}、{firstname}）
- ✅ 真正并发发送（多账号同时运行）
- ✅ 并发控制（线程数+启动间隔）
- ✅ 额度限制（单账号/总上限）
- ✅ 随机间隔（防检测）
- ✅ 遇到限制自动切换账号
- ✅ 从采集/文件导入目标用户

### 👥 采集用户
- ✅ 从群组采集成员
- ✅ 从频道采集订阅者
- ✅ 过滤条件设置（活跃用户/排除机器人/仅有用户名）
- ✅ 采集数量限制
- ✅ 导出到私信广告
- ✅ 保存为文件

## 🚀 使用方法

### 快速开始

```bash
python main_v1.3.py
```

### 使用流程

#### 1. 账号管理

1. 点击「📂 账号管理」标签
2. 点击「📁 导入 Session 文件夹」
3. 选择存放 `.session` 文件的文件夹
4. 点击「🔍 检测状态」检测账号
5. 双击账号切换选中/取消
6. 或使用「✅ 全选」/「❌ 清空」按钮

#### 2. 采集用户（可选）

1. 点击「👥 采集用户」标签
2. 选择采集来源（群组/频道）
3. 输入目标链接（例如：https://t.me/group_name 或 @channel_name）
4. 点击「➕ 添加到列表」
5. 设置过滤条件
6. 点击「🚀 开始」采集
7. 双击用户切换选中
8. 点击「📤 导出到私信广告」

#### 3. 私信广告

1. 点击「✉️ 私信广告」标签
2. 确认已选账号数量（从账号管理）
3. 选择发送类型（文本消息/转发贴子）
4. 输入消息内容
5. 输入目标用户（或从采集导入）
6. 配置发送参数
   - 并行线程数：同时发送的账号数量（推荐 2-5）
   - 启动间隔：每批账号启动间隔（推荐 1-3 秒）
   - 单账号上限：每个账号最多发送数量（推荐 50）
   - 发送间隔：每条消息间隔时间（推荐 3-8 秒）
7. 点击「🚀 开始」发送

## ⚙️ 配置说明

### 并发设置

- **并行线程数**：建议 2-5，过多可能被 Telegram 检测
- **启动间隔**：建议 1-3 秒，避免同时登录

### 额度限制

- **单账号上限**：每个账号最多发送数量，防止单号过载
- **任务总上限**：整个任务最多发送数量，控制总量

### 发送间隔

- **间隔范围**：每条消息间隔时间，建议 3-10 秒
- **随机间隔**：在范围内随机，更像真人行为

## 📝 变量替换

在文本消息中可以使用以下变量：

- `{username}` - 目标用户的用户名
- `{firstname}` - 目标用户的名字

示例：

```
你好 {firstname}！
欢迎关注 @{username}
```

## ⚠️ 注意事项

1. **Session 文件安全**
   - Session 文件包含登录凭证，请妥善保管
   - 不要泄露给他人

2. **频率限制**
   - Telegram 有严格的反 spam 机制
   - 建议控制发送速度，避免被封号
   - 建议开启「遇到限制自动切换账号」

3. **隐私保护**
   - 部分用户设置了隐私限制，无法接收陌生人消息
   - 程序会自动跳过这些用户

4. **合法使用**
   - 仅用于合法营销和通知
   - 不要发送垃圾信息或违法内容

## 🐛 常见问题

### Q: 提示"Invalid API key"

A: 程序使用 Telegram Desktop 官方 API，不需要自己申请。如果提示此错误，可能是网络问题。

### Q: 发送速度太慢

A: 可以适当增加"并行线程数"，但不要超过 10，否则可能被检测。

### Q: 账号被限制发送消息

A: 可能触发了 Telegram 的反 spam 机制，建议：
- 减少发送速度
- 增加发送间隔
- 分批发送
- 开启自动切换账号

### Q: 并发不生效，还是顺序执行

A: 请确认使用的是 v1.3 版本，旧版本（v1.0/v1.1）是顺序执行。

### Q: 采集用户失败

A: 可能原因：
- 账号没有加入该群组/频道
- 群组设置了隐私保护
- 网络问题
